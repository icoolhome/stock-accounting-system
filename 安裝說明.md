# 股票記帳系統 - 安裝與使用說明

## 📋 前置需求

- **Node.js** 18.0 或更高版本
- **npm**（通常隨 Node.js 一起安裝）

## 🚀 快速開始

### 步驟 1：安裝依賴

在專案根目錄執行：

**方法 1：使用 setup.bat（推薦）**
```
雙擊執行 setup.bat
```

**方法 2：使用命令列**
```bash
npm run install:all
```

這會自動安裝根目錄、server 和 client 的所有依賴套件。

### 步驟 2：啟動系統

在專案根目錄執行：

**方法 1：使用 start.bat（推薦）**
```
雙擊執行 start.bat
```

**方法 2：使用命令列**
```bash
npm run dev
```

## 🌐 系統網址與端口

啟動系統後，會開啟以下服務：

| 服務 | 網址 | 端口 | 說明 |
|------|------|------|------|
| **前端應用** | http://localhost:3000 | **3000** | 使用者介面（主要訪問網址） |
| **後端 API** | http://localhost:3001 | **3001** | 後端服務器（API 服務） |

### ⚠️ 重要：初始訪問網址

**請在瀏覽器中訪問：**
```
http://localhost:3000
```

這是系統的主要入口網址。

## 🪟 啟動後的窗口說明

使用 `start.bat` 啟動後，會開啟以下窗口：

### ✅ 必須保持開啟的窗口

1. **「股票記帳系統 - 伺服器」窗口**
   - **功能**：後端 API 服務器
   - **端口**：3001
   - **狀態提示**：顯示 "伺服器運行在 http://localhost:3001"
   - **⚠️ 不能關閉**：關閉後前端將無法連接 API

2. **「股票記帳系統 - 客戶端」窗口**
   - **功能**：前端預覽服務器
   - **端口**：3000
   - **狀態提示**：顯示 "Local: http://localhost:3000/"
   - **⚠️ 不能關閉**：關閉後無法訪問網頁應用

### ℹ️ 可以關閉的窗口

- **主啟動窗口**：啟動完成後可以關閉（服務器會繼續在各自的窗口中運行）

## 📝 使用流程

1. **啟動系統**
   - 執行 `start.bat`
   - 等待兩個服務器窗口都啟動完成

2. **訪問系統**
   - 瀏覽器會自動打開 http://localhost:3000
   - 如果沒有自動打開，請手動在瀏覽器中輸入上述網址

3. **登入系統**
   - 首次使用需要註冊帳號
   - 註冊後使用郵箱和密碼登入

4. **使用系統**
   - 確保兩個服務器窗口保持開啟
   - 可以開始使用各項功能

## 🛑 停止系統

要停止系統，請：

1. **關閉「股票記帳系統 - 伺服器」窗口**
   - 在窗口標題欄點擊關閉按鈕（X）
   - 或按 `Ctrl+C` 後再關閉

2. **關閉「股票記帳系統 - 客戶端」窗口**
   - 在窗口標題欄點擊關閉按鈕（X）
   - 或按 `Ctrl+C` 後再關閉

## ⚠️ 常見問題

### Q: 網頁無法打開怎麼辦？

**A: 請檢查：**

1. ✅ 兩個服務器窗口是否都還在運行？
   - 「股票記帳系統 - 伺服器」窗口必須開啟
   - 「股票記帳系統 - 客戶端」窗口必須開啟

2. ✅ 訪問的網址是否正確？
   - 正確：http://localhost:3000
   - 錯誤：http://localhost:3001（這是後端 API，不是前端）

3. ✅ 端口是否被占用？
   - 如果端口被占用，請關閉占用端口的程序或修改端口配置

### Q: 提示 "連接失敗" 或 "無法連接 API"？

**A: 請確認後端服務器窗口是否正常運行**

- 檢查「股票記帳系統 - 伺服器」窗口是否顯示 "伺服器運行在 http://localhost:3001"
- 如果窗口已關閉，請重新執行 `start.bat`

### Q: 如何確認系統正常運行？

**A: 檢查以下項目：**

1. ✅ 兩個服務器窗口都正常開啟
2. ✅ 瀏覽器可以訪問 http://localhost:3000
3. ✅ 可以看到登入頁面
4. ✅ 可以正常註冊和登入
